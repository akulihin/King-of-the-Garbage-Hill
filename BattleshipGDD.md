## 1. Глоссарий

| Термин | Определение |
|--------|-------------|
| **Палуба** (deck) | Секция корабля со своей прочностью. Корабль состоит из 1–4 палуб. |
| **Прочность** (armor) | Сколько урона выдерживает палуба до уничтожения. По умолчанию = 2. |
| **Space** | Радиус клеток вокруг корабля, в котором нельзя ставить другие корабли. По умолчанию = 1, максимум = 2. Также используется для пассивок (заморозка, разведка, просветка при убийстве). |
| **Класс дальности** | Определяет правила размещения и стрельбы корабля: Close, Close (melee), Mid, Tetra, Far. |
| **Активка** | Особый выстрел (не обычная баллиста), использует ход вместо обычного выстрела. |
| **Сумон** | Призываемый юнит на вражеское поле. Призыв не тратит ход. |
| **Выстрел** (shot) | Единица времени. Считаются суммарно для обоих игроков. |
| **Ход** (turn) | Очередь игрока. Может включать несколько выстрелов при ресетах. |
| **Ресет хода** | Продолжение хода после уничтожения вражеской палубы. |
| **Просвеченная клетка** (revealed cell) | Клетка на вражеском поле, содержимое которой было раскрыто. Максимум = 100. |
| **Домашний юнит** | Корабль, который можно использовать только на определённой половине поля. |
| **Burn** | Статус — мгновенное сгорание всего корабля. |
| **Freeze** | Статус — заморозка, выключает корабль/сумона из игры. |
| **Devastated** | Статус — опустошение, выключает корабль из игры (аналог freeze, другая анимация; разведчик не передаёт информацию). |
| **Capture** | Статус — захват корабля, он начинает считаться вражеским. |
| **Destroy** | Статус — уничтожение палубы с выключением модуля на ней. |
| **Stun** (оглушение) | Статус — пропуск хода, срабатывает в течение 1–2 выстрелов. |
| **Penalty** (штраф) | Статус — пропуск следующего хода, без срока давности. |
| **Burn resist** | Статус — иммунитет к сгоранию. |

## 2. Обзор

### Core Loop

1. **Выбор армии** — Империя или Альянс (WIP).
2. **Расстановка и апгрейды** — размещение кораблей на поле 10x10 + распределение 40 монет на улучшения.
3. **Бой** — пошаговая стрельба с ресетами, активками и сумонами.
4. **Финальный абордаж** — когда у одного игрока кончаются корабли класса Mid.

### Фракции

- **Империя** — основная армия с полным набором юнитов.
- **Альянс** — WIP.
- **Общие** — доступны всем армиям.

### Регионы

4 региона: Юг, Запад, Север, Восток. Игрок может использовать корабли максимум из **3 регионов из 4**. Регион **Тетракор** — отдельный, не учитывается в ограничении.

## 3. Основные механики

### 3.1. Структура хода

```
Начало хода игрока
├─ Призыв сумона (опционально, не тратит ход, макс 1 за 2 выстрела)
├─ Выбор действия:
│  ├─ Есть корабли класса Mid или Close?
│  │  ├─ Да → Обычный выстрел доступен
│  │  └─ Нет → Обычный выстрел НЕ доступен
│  ├─ Есть снаряды у активок?
│  │  ├─ Да → Активка доступна
│  │  └─ Нет → Активка НЕ доступна
│  └─ Ничего не доступно → проверка условий поражения (см. раздел 10)
├─ Выстрел по вражеской карте
│  ├─ Попадание в палубу вражеского корабля
│  │  ├─ Палуба уничтожена → РЕСЕТ (стреляй ещё)
│  │  └─ Палуба выжила (броня > 0) → ХОД ПРЕРВАН ("Поцарапал броню!")
│  ├─ Попадание в сумона → ХОД ПРЕРВАН (+ штраф, см. «Штраф при убийстве сумона»)
│  └─ Промах (пустая клетка) → ХОД ПЕРЕДАН врагу
└─ Выстрел по своей карте (например, убийство вражеского сумона на своём поле)
   └─ ХОД ПРЕРВАН (+ штраф, см. «Штраф при убийстве сумона»)
```

Пауза между выстрелами: несколько секунд при попадании. При промахе — без паузы.

### 3.2. Урон и броня

- Урон обычной баллисты = **2**.
- Прочность обычной палубы = **2**.
- При остатке прочности > 0 — "Поцарапал броню!", ход прерывается. Палуба закрашивается особым цветом (коричневый).
- При прочности = 0 — палуба уничтожена.
- Броня не уходит ниже нуля.
- Все значения — целые числа, максимум 9.

**Множители урона:**

| Множитель | Значение |
|-----------|----------|
| Белый камень | x4 |
| Дробь | x0.5 |

Множитель урона — модифицируемый параметр в коде для удобной балансировки.

**Виды брони:** текущая броня и первоначальная броня — отслеживаются отдельно (для будущих механик).

Визуально цифры не отображаются — играем без чисел на экране.

### 3.3. Дальность

| Класс | Размещение | Стрельба | Абордаж | Примечания |
|-------|-----------|----------|---------|------------|
| **Close** | Строки 1–8 | Обычные выстрелы (пока есть Mid или Close) | Да — идёт в финальный абордаж | Наличие Close позволяет делать обычные выстрелы |
| **Close (melee)** | Строки 1–8 | 0 снарядов, не стреляет вообще | Да — идёт в финальный абордаж | Если остались только Close (melee) — обычные выстрелы невозможны |
| **Mid** | Строки 1–8 | Обычные выстрелы | Нет | Наличие Mid позволяет делать обычные выстрелы. Конец Mid → финальный абордаж |
| **Tetra** | Где угодно, включая строки 9–10 | Активки (снаряды) | Нет | Палубы не выходят на 7–8 строки при постановке на 9–10. Артиллерия |
| **Far** | Где угодно, включая строки 9–10 | Активки (снаряды), но с 9–10 строки не достаёт до 9–10 вражеского поля | Нет | Прицел красный = не выстрелить. Артиллерия |

Tetra и Far — оба считаются **артиллерией**.

### 3.4. Передвижение

Три типа передвижения:

1. **Абордаж** — сумоны и абордажные корабли ставятся на 1-ю строку вражеского поля. Двигаются вниз, проходя за каждый выстрел столько клеток, сколько имеют скорости. Достигнув конца карты — через 1 выстрел можно пустить обратно снизу вверх по соседнему ряду.

2. **Автоматическое** — Тройка из лёгкого дерева. Если выстрел летит в нос или корму — корабль автоматически отплывает в противоположную сторону (если есть место). Работает пока есть место. Корабль заплывает в ловушки — игрок не может остановить.

3. **Мануальное** — Маневрирующая двойка. После убийства палубы — может отплыть на 1–2 клетки вперёд или назад (если есть место). Решает игрок. Отплыть можно только в начале **своего** хода. Работает один раз за бой.

### 3.5. Просветка клеток

| Действие | Просветка |
|----------|-----------|
| Промах | Просвечивается клетка промаха |
| Попадание в палубу | Просвечивается клетка попадания |
| Убийство/захват корабля | Просвечиваются все клетки в Space вокруг корабля |
| Разведчик | Отложенная просветка (информация выдаётся после гибели/выхода разведчика) |

Повторная просветка уже раскрытой клетки **не увеличивает** счётчик. Максимум = 100.

**Просветка ≠ актуальная позиция.** Если корабль передвинулся после просветки — отметки на карте остаются на старых местах. Попадание отображается там, где было попадание, промах — там, где был промах, даже если корабль уже сместился. Реальная позиция раскрывается только при убийстве корабля.

## 4. Статусы и взаимодействия

### Определения статусов

| Статус | Эффект |
|--------|--------|
| **Burn** | Весь корабль мгновенно сгорает (все палубы уничтожены). Попадание горючкой даже в уже повреждённую палубу сжигает весь корабль. |
| **Burn resist** | Иммунитет к burn. Переносная механика (не привязана к одному кораблю). |
| **Freeze** | Выключает корабль/сумона из игры (эквивалент смерти). Клетки вокруг просвечиваются. Срабатывает даже при столкновении — урон не наносится. Разведчик при freeze **не** передаёт информацию. |
| **Devastated** | Аналог freeze, но с другой анимацией. Выключает корабль из игры. Разведчик при devastated **не** передаёт информацию. |
| **Capture** | Корабль считается вражеским. Его необходимо убить для победы. Противник может стрелять **только** в захваченный корабль, пока статус активен. Capture перебивает вражеский Capture (рекапчер). Условие: корабль должен существовать и не быть уничтоженным. |
| **Destroy** | Уничтожает палубу и выключает модуль на ней. Особый цвет закраски. |
| **Stun** | Пропуск хода. Срабатывает на следующий или через один выстрел. Исчезает после срабатывания или истечения времени. |
| **Penalty** | Пропуск следующего хода. Без срока давности, исчезает после срабатывания. |

### Матрица взаимодействий

| Источник | Цель | Результат |
|----------|------|-----------|
| Burn | Корабль с burn resist | Нет эффекта |
| Burn | Пираты / Проклятый пират | Корабль сгорает, сумон **не** выпускается |
| Freeze (Драккар) | Любой сумон в Space | Сумон уничтожен, урон не нанесён |
| Freeze (Драккар) | Абордажный корабль в Space | Корабль уничтожен, урон не нанесён (даже при появлении в клетке Драккара) |
| Capture (Пиратская лодка) | 1–2 палубный корабль | Корабль захвачен |
| Capture (Пиратская лодка) | 3–4 палубный корабль | Нет эффекта, лодка погибает |
| Devastated (Проклятая лодка) | Любой корабль | Корабль выключен из игры |
| Горючая баржа получает урон | Корабли в Space = 2 | Все корабли в зоне сгорают (burn), включая союзные |

## 7. Юниты — Сумоны

### Общие правила сумонов

- **4 слота** для сумонов (без учёта спец. кораблей с собственными сумонами — для них отдельный слот).
- **Условие призыва:** 5 просвеченных клеток на вражеской карте за каждого сумона (2-й сумон = 10, 3-й = 15, 4-й = 20).
- **Прочность** = 1 (ваншот).
- **Призыв не тратит ход.** Можно призвать в любой момент, даже посреди вражеского хода.
- Максимум **1 сумон за 2 выстрела**.
- Сумон появляется на любой клетке **1-й строки** вражеского поля.
- Движение: начинает двигаться после следующего выстрела (призвал → стоит → выстрел → передвинулся).
- Двигается сверху вниз, проходя за каждый выстрел столько клеток, сколько имеет скорости.
- **Достигнув конца карты:** через 1 выстрел можно пустить обратно снизу вверх по соседнему ряду.

### Штраф при убийстве сумона

| Ситуация | Результат |
|----------|-----------|
| Убийство на **1–3 строках** | Враг получает **Penalty** (штраф) |
| Убийство при **появлении** (до движения) | Ход прерван, штрафа **нет** |
| Убийство на **4+ строках** | Ход прерван, штрафа **нет** |

## 9. Финальный абордаж

### Триггер

Когда у одного из игроков **кончаются корабли класса Mid** — этот игрок получает статус **Final Rush**.

### Механика

1. Все корабли игрока классов **Close** и **Close (melee)** превращаются в активки.
2. Их необходимо расставить **на одном ходу** на любую клетку 1-й строки вражеского поля (как сумонов).
3. Передвигаются как сумоны — сверху вниз, проходя за каждый выстрел столько клеток, сколько имеют скорости.
4. Просвечивают клетки в радиусе Space.
5. Достигнув конца карты — через 1 выстрел можно послать снизу вверх по соседнему ряду.
6. Уязвимы как сумоны (ваншот).

### Урон при таране

| Цель | Результат |
|------|-----------|
| 1–2 палубный корабль | Накладывает **Devastated**, плывёт дальше |
| 3–4 палубный корабль | Урон от тарана = 4, абордажный корабль **погибает** |

### Дополнительно

- Все оставшиеся **сумоны** тоже идут в абордаж.
- При покупке апгрейда "Экипаж" — Трёшки выпускают сумона "Пиратская лодка".
- Тетракамнемёт получает дополнительный снаряд Белый камень при начале абордажа.
- Горючка получает дополнительный снаряд при абордаже.

## 10. Условия победы и поражения

### Победа

1. Уничтожить **все** вражеские корабли (не считая сумонов).
2. Уничтожить все вражеские корабли, **кроме артиллерии** (Tetra и Far), при условии что их оставшихся снарядов **не хватит** для уничтожения оставшихся палуб.

### Поражение

1. Все корабли уничтожены.
2. Не осталось кораблей класса Mid или Close, нет Close (melee), и у артиллерии (Tetra и Far) оставшихся снарядов < вражеских оставшихся палуб.

### Особые случаи

- **Иссушитель:** если начинается абордаж от любой стороны — автоматическая победа владельца Иссушителя. Если у обоих игроков по Иссушителю — пассивка выключается.
- **Capture:** захваченный корабль считается вражеским и его нужно уничтожить для победы.

## 12. Пропуски хода

### Stun (оглушение)

Если на игрока наложен stun, он пропустит ход, если этот ход начнётся на **следующий** или **через один выстрел** после получения стана. Эффект пропадает после срабатывания или истечения.

**Примеры:**
1. Станящий выстрел попал в палубу → ресет → промах → **враг пропускает** (через 1 выстрел).
2. Станящий выстрел промахнулся → ход врага → **сразу отменяется**, ход возвращается (на следующий выстрел).
3. Станящий выстрел попал → ресет → **снова** попал → стан уже рассеялся.

### Penalty (штраф)

Игрок пропустит **следующий ход**, независимо от того когда он наступит — хоть через сто выстрелов. Эффект пропадает после срабатывания.

Технически stun и penalty — право на бесплатный промах для врага, не сбивающий серию атак/ресетов.
